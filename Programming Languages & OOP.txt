1::What is Object Oriented Programming?::A programming paradigm based on objects and classes.::OOP models real-world entities using data and behavior together.
2::What is an object?::An instance of a class.::Objects represent real-world entities with state and behavior.
3::What is a class?::A blueprint for creating objects.::Defines properties and methods.
4::What are the four pillars of OOP?::Encapsulation, Abstraction, Inheritance, Polymorphism.::They define core OOP behavior.
5::What is encapsulation?::Binding data and methods together.::Protects data using access control.
6::How is encapsulation achieved?::Using access specifiers.::Controls visibility of members.
7::What is abstraction?::Hiding implementation details.::Shows only essential features.
8::How is abstraction implemented?::Using abstract classes or interfaces.::Forces implementation of required methods.
9::What is inheritance?::Acquiring properties of another class.::Promotes code reuse.
10::What is polymorphism?::One interface, multiple implementations.::Allows method overriding and overloading.
11::What is compile-time polymorphism?::Method overloading.::Resolved at compile time.
12::What is runtime polymorphism?::Method overriding.::Resolved at runtime.
13::What is method overloading?::Same method name, different parameters.::Improves readability.
14::What is method overriding?::Redefining parent class method.::Achieves runtime polymorphism.
15::What is dynamic binding?::Method call resolved at runtime.::Supports polymorphism.
16::What is static binding?::Method call resolved at compile time.::Occurs with overloading.
17::What is an interface?::A contract with abstract methods.::Supports multiple inheritance.
18::What is an abstract class?::Class with abstract methods.::Cannot be instantiated.
19::Difference between abstract class and interface?::Abstract class can have implementation; interface cannot.::Design flexibility difference.
20::What is constructor?::Special method to initialize object.::Runs automatically during object creation.
21::What is destructor?::Method to free resources.::Called when object is destroyed.
22::What is default constructor?::Constructor with no parameters.::Provides default initialization.
23::What is parameterized constructor?::Constructor with parameters.::Initializes object with values.
24::What is copy constructor?::Creates object from another object.::Used in C++ for deep copying.
25::What is constructor overloading?::Multiple constructors in a class.::Allows different initialization.
26::What is this keyword?::Reference to current object.::Resolves naming conflicts.
27::What is super keyword?::Refers to parent class.::Accesses superclass members.
28::What is access specifier?::Defines member visibility.::Examples: public, private, protected.
29::What is private access?::Accessible within class only.::Ensures data security.
30::What is protected access?::Accessible in same package or subclass.::Supports inheritance.
31::What is public access?::Accessible everywhere.::No restrictions.
32::What is data hiding?::Restricting direct data access.::Achieved via encapsulation.
33::What is data abstraction?::Exposing only relevant details.::Simplifies usage.
34::What is multiple inheritance?::Inheriting from multiple classes.::Supported in C++ via interfaces.
35::Why Java does not support multiple inheritance?::To avoid diamond problem.::Ambiguity prevention.
36::What is diamond problem?::Ambiguity in multiple inheritance.::Occurs when two parents have same method.
37::What is virtual function?::Function overridden in derived class.::Supports runtime polymorphism.
38::What is virtual destructor?::Ensures proper object cleanup.::Important in inheritance.
39::What is base class?::Parent class.::Provides common functionality.
40::What is derived class?::Child class.::Inherits from base class.
41::What is object slicing?::Loss of derived object data.::Occurs in C++ assignment.
42::What is final keyword?::Prevents modification.::Used for classes and methods.
43::What is static keyword?::Belongs to class, not object.::Shared among instances.
44::What is static method?::Method without object instance.::Called using class name.
45::What is static variable?::Shared variable across objects.::Single memory allocation.
46::What is const keyword?::Prevents value modification.::Ensures immutability.
47::What is immutable object?::Object whose state cannot change.::Examples include String in Java.
48::What is deep copy?::Copies actual object data.::Independent memory allocation.
49::What is shallow copy?::Copies object references.::Shared memory risk.
50::What is object cloning?::Creating exact object copy.::Uses clone method.
51::What is garbage collection?::Automatic memory management.::Reclaims unused memory.
52::What is memory leak?::Unused memory not released.::Leads to performance issues.
53::What is heap memory?::Dynamic memory allocation area.::Objects stored here.
54::What is stack memory?::Function call storage.::Fast access memory.
55::What is composition?::Has-a relationship.::Stronger than aggregation.
56::What is aggregation?::Weak has-a relationship.::Objects exist independently.
57::Difference between composition and inheritance?::Composition is flexible.::Favors loose coupling.
58::What is association?::General relationship between objects.::No ownership.
59::What is UML?::Unified Modeling Language.::Used for system design.
60::What is class diagram?::UML representation of classes.::Shows attributes and methods.
61::What is object diagram?::Runtime instance representation.::Shows object relationships.
62::What is sequence diagram?::Interaction flow diagram.::Shows message sequence.
63::What is dependency?::One class uses another.::Temporary relationship.
64::What is cohesion?::Degree of module focus.::High cohesion is desirable.
65::What is coupling?::Interdependency between modules.::Low coupling is better.
66::What is tight coupling?::High dependency.::Hard to maintain.
67::What is loose coupling?::Low dependency.::Improves flexibility.
68::What is SOLID?::Five OOP design principles.::Improves maintainability.
69::What is Single Responsibility Principle?::One reason to change.::Each class has one job.
70::What is Open-Closed Principle?::Open for extension, closed for modification.::Supports scalability.
71::What is Liskov Substitution Principle?::Subclasses replace base class.::Ensures correct inheritance.
72::What is Interface Segregation Principle?::Small focused interfaces.::Avoids unused methods.
73::What is Dependency Inversion Principle?::Depend on abstractions.::Reduces coupling.
74::What is design pattern?::Reusable design solution.::Solves common problems.
75::What is Singleton pattern?::Single instance class.::Controls object creation.
76::What is Factory pattern?::Object creation abstraction.::Decouples instantiation.
77::What is Abstract Factory?::Factory of factories.::Creates related objects.
78::What is Builder pattern?::Step-by-step object creation.::Handles complex objects.
79::What is Prototype pattern?::Object cloning pattern.::Avoids costly creation.
80::What is Adapter pattern?::Interface compatibility pattern.::Bridges incompatible interfaces.
81::What is Decorator pattern?::Adds behavior dynamically.::Extends functionality.
82::What is Observer pattern?::One-to-many dependency.::Used in event handling.
83::What is Strategy pattern?::Encapsulated algorithms.::Runtime behavior selection.
84::What is MVC?::Model-View-Controller architecture.::Separates concerns.
85::What is exception?::Runtime error.::Disrupts normal flow.
86::What is exception handling?::Managing runtime errors.::Improves reliability.
87::What is try-catch block?::Exception handling construct.::Catches errors.
88::What is finally block?::Always executed block.::Used for cleanup.
89::What is checked exception?::Compile-time exception.::Must be handled.
90::What is unchecked exception?::Runtime exception.::Optional handling.
91::What is polymorphic behavior?::Different responses to same call.::Core OOP benefit.
92::What is virtual table?::Function lookup table.::Used in runtime polymorphism.
93::What is RTTI?::Run-Time Type Information.::Identifies object type.
94::What is namespace?::Logical grouping of identifiers.::Avoids name conflicts.
95::What is friend function?::Access private members.::C++ specific feature.
96::What is operator overloading?::Redefining operators.::Improves readability.
97::What is method hiding?::Static method redefinition.::Occurs in Java.
98::What is shadowing?::Variable name conflict.::Inner scope hides outer.
99::What is inner class?::Class within class.::Improves encapsulation.
100::What is anonymous class?::Class without name.::Used for one-time use.
101::What is lambda expression?::Anonymous function.::Supports functional programming.
102::What is interface default method?::Method with body in interface.::Java 8 feature.
103::What is multiple interface inheritance?::Implementing many interfaces.::Allowed in Java.
104::What is virtual base class?::Prevents duplication.::Solves diamond problem.
105::What is abstract method?::Method without body.::Must be overridden.
106::What is final method?::Cannot be overridden.::Ensures behavior stability.
107::What is constructor chaining?::Calling one constructor from another.::Improves reuse.
108::What is destructor chaining?::Calling destructors in inheritance.::Ensures cleanup.
109::What is object lifetime?::Duration object exists.::Managed by scope or GC.
110::What is memory encapsulation?::Restricting memory access.::Improves safety.
111::What is polymorphic pointer?::Base class pointer to derived object.::Enables dynamic binding.
112::What is covariant return type?::Derived return allowed.::Java and C++ feature.
113::What is early binding?::Compile-time binding.::Faster execution.
114::What is late binding?::Runtime binding.::Flexible behavior.
115::What is abstract data type?::Data + operations.::Encapsulation principle.
116::What is interface-based programming?::Coding to interfaces.::Improves flexibility.
117::What is method signature?::Method name + parameters.::Defines uniqueness.
118::What is object serialization?::Convert object to byte stream.::Used in storage/network.
119::What is deserialization?::Recreating object from stream.::Reverse of serialization.
120::What is transient keyword?::Excludes field from serialization.::Java feature.
121::What is volatile keyword?::Thread visibility guarantee.::Concurrency support.
122::What is multithreading?::Multiple threads execution.::Improves performance.
123::What is synchronization?::Thread safety mechanism.::Prevents race conditions.
124::What is deadlock?::Threads waiting indefinitely.::Caused by resource lock.
125::What is race condition?::Unpredictable behavior.::Occurs without synchronization.
126::What is inheritance vs composition?::Composition preferred.::More flexible design.
127::What is object responsibility?::Assigned behavior.::Defined in SRP.

131::What is lazy loading?::Load when needed.::Optimizes memory.
132::What is class loader?::Loads classes at runtime.::Java mechanism.
133::What is reflection?::Runtime class inspection.::Used in frameworks.
134::What is dependency injection?::Providing dependencies externally.::Improves testability.
135::What is inversion of control?::Framework controls flow.::Core of DI.
136::What is object pool?::Reusable object collection.::Improves performance.
137::What is command pattern?::Encapsulates requests.::Supports undo/redo.
138::What is facade pattern?::Simplified interface.::Hides complexity.
