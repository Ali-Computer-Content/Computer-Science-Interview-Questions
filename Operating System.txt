1::What is an Operating System?::An OS is system software that manages hardware and provides services to applications::It acts as an intermediary between user programs and computer hardware
2::Main functions of an Operating System?::Process management, memory management, file management, device management, security::These core functions ensure efficient and secure system operation
3::What is a process?::A process is a program in execution::It includes program code, data, stack, and CPU state
4::Difference between process and program?::Program is passive, process is active::A program becomes a process when loaded into memory and executed
5::What is a thread?::A lightweight process that shares memory with other threads::Threads improve performance through parallelism
6::Process states in OS?::New, Ready, Running, Waiting, Terminated::These states describe a process lifecycle
7::What is context switching?::Saving and restoring CPU state between processes::Allows multitasking on a single CPU
8::What is CPU scheduling?::Selecting a process from ready queue for execution::Ensures efficient CPU utilization
9::FCFS scheduling disadvantage?::Convoy effect::Long processes delay short ones
10::What is SJF scheduling?::Shortest Job First scheduling algorithm::Minimizes average waiting time
11::What is preemptive scheduling?::CPU can be taken from running process::Used in Round Robin and SRTF
12::What is Round Robin scheduling?::Time-sliced CPU scheduling algorithm::Ensures fairness among processes
13::What is starvation?::Indefinite postponement of a process::Occurs in priority scheduling
14::What is aging?::Increasing priority over time::Prevents starvation
15::What is deadlock?::Processes waiting indefinitely for resources::Occurs due to circular wait
16::Four conditions for deadlock?::Mutual exclusion, hold and wait, no preemption, circular wait::All must hold simultaneously
17::Deadlock prevention method?::Break at least one deadlock condition::Example: resource ordering
18::What is deadlock avoidance?::Ensure system never enters unsafe state::Uses Banker's Algorithm
19::What is Banker's Algorithm?::Deadlock avoidance algorithm::Checks safe sequence before allocation
20::Difference between deadlock and starvation?::Deadlock is circular wait, starvation is indefinite delay::Starvation may resolve eventually
21::What is semaphore?::Synchronization primitive using counters::Controls access to shared resources
22::Binary semaphore value?::0 or 1::Used as mutex
23::What is mutex?::Mutual exclusion lock::Prevents race conditions
24::What is critical section?::Code accessing shared resources::Must be executed exclusively
25::Race condition occurs when?::Multiple processes access shared data concurrently::Leads to inconsistent results
26::What is monitor?::High-level synchronization construct::Encapsulates shared data and operations
27::What is memory management?::Managing RAM allocation to processes::Ensures efficient memory utilization
28::What is paging?::Memory divided into fixed-size pages::Eliminates external fragmentation
29::What is page fault?::Requested page not in memory::OS loads it from disk
30::What is segmentation?::Memory divided based on logical units::Supports user view of memory
31::Difference between paging and segmentation?::Paging is fixed size, segmentation is variable::Paging avoids fragmentation
32::What is virtual memory?::Using disk as extension of RAM::Allows large programs to run
33::What is thrashing?::System spends more time paging than executing::Occurs when memory is overcommitted
34::What is LRU page replacement?::Least Recently Used page removed::Based on temporal locality
35::What is FIFO page replacement drawback?::Beladyâ€™s anomaly::More frames can cause more faults
36::What is file system?::Manages data storage and retrieval::Provides abstraction over storage devices
37::File access methods?::Sequential, Direct, Indexed::Defines how files are read/written
38::What is inode?::Metadata structure in Unix::Stores file attributes and pointers
39::Difference between absolute and relative path?::Absolute starts from root, relative from current directory::Defines file location
40::What is disk scheduling?::Order of disk I/O requests execution::Reduces seek time
41::SCAN disk scheduling is also called?::Elevator algorithm::Moves disk arm back and forth
42::What is RAID?::Redundant Array of Independent Disks::Improves performance and reliability
43::What is RAID 1?::Disk mirroring::Provides fault tolerance
44::What is RAID 5?::Block-level striping with parity::Balances speed and redundancy
45::What is booting?::Loading OS into memory::Occurs at system startup
46::Difference between cold and warm boot?::Cold starts power off, warm restarts system::Warm is faster
47::What is kernel?::Core of OS::Manages system resources
48::Monolithic kernel example?::Linux::All services run in kernel space
49::Microkernel advantage?::Better reliability and security::Minimal kernel functionality
50::User mode vs Kernel mode?::Restricted vs full hardware access::Prevents system crashes
51::What is multiprogramming?::Keeping multiple programs in memory simultaneously::Multiprogramming improves CPU utilization by ensuring CPU always has a job to execute
52::Difference between multiprogramming and multitasking?::Multiprogramming switches jobs, multitasking switches tasks rapidly::Multitasking focuses on user interaction while multiprogramming maximizes CPU usage
53::What is real-time operating system?::OS with strict timing constraints::RTOS guarantees response within a fixed deadline
54::Hard vs Soft real-time OS?::Hard cannot miss deadlines, Soft can tolerate delays::Hard RTOS used in medical and avionics systems
55::What is batch operating system?::Executes jobs in batches without interaction::Used in early mainframe systems
56::What is time-sharing OS?::Allows multiple users to share CPU time::Ensures quick response to each user
57::What is distributed operating system?::OS managing multiple computers as one system::Provides transparency and resource sharing
58::What is symmetric multiprocessing?::Multiple CPUs share same memory::Improves performance and reliability
59::What is asymmetric multiprocessing?::One master CPU controls others::Simpler but less efficient than SMP
60::What is inter-process communication?::Mechanism for processes to communicate::Includes pipes, message queues, shared memory
61::What is pipe?::Unidirectional communication channel::Commonly used between parent-child processes
62::Difference between named and unnamed pipe?::Named exists in filesystem, unnamed does not::Named pipes allow unrelated process communication
63::What is message queue?::Message-based IPC mechanism::Processes exchange structured messages
64::What is shared memory?::Memory shared among processes::Fastest IPC method but requires synchronization
65::Why synchronization is required?::To avoid data inconsistency::Ensures correct execution of concurrent processes
66::What is spinlock?::Busy-wait locking mechanism::CPU keeps checking until lock is released
67::What is priority inversion?::Low-priority process blocks high-priority process::Occurs due to resource locking
68::What is priority inheritance?::Temporarily raising priority of low-priority process::Solves priority inversion problem
69::What is memory fragmentation?::Wasted memory space::Occurs due to inefficient allocation
70::Internal fragmentation occurs when?::Allocated memory larger than requested::Common in fixed partition systems
71::External fragmentation occurs when?::Free memory scattered in small blocks::Occurs in variable partition systems
72::What is compaction?::Combining free memory blocks::Reduces external fragmentation
73::What is swapping?::Moving processes between memory and disk::Used to free RAM
74::Difference between swapping and paging?::Swapping moves whole process, paging moves pages::Paging is more efficient
75::What is demand paging?::Load pages only when required::Reduces memory usage
76::What is page table?::Maps virtual pages to physical frames::Used for address translation
77::What is TLB?::Translation Lookaside Buffer::Caches recent page table entries
78::What happens on TLB miss?::Page table is accessed::Slower memory access occurs
79::What is address binding?::Mapping logical to physical address::Can be compile-time, load-time, or run-time
80::Logical vs Physical address?::Logical generated by CPU, physical by memory unit::MMU converts logical to physical
81::What is MMU?::Memory Management Unit::Handles address translation
82::What is disk seek time?::Time to move disk arm::Major factor in disk performance
83::What is rotational latency?::Time for disk rotation::Delay before sector reaches head
84::What is disk throughput?::Number of requests handled per unit time::Higher throughput improves performance
85::What is SSTF disk scheduling?::Shortest Seek Time First::Minimizes seek time but causes starvation
86::C-SCAN advantage?::Uniform wait time::Improves fairness over SCAN
87::What is file allocation table?::Table storing file block locations::Used in FAT file systems
88::Contiguous file allocation disadvantage?::External fragmentation::File size must be known in advance
89::Linked file allocation disadvantage?::Slow access::Each block points to next
90::Indexed allocation advantage?::Direct access supported::Uses index block
91::What is file descriptor?::Integer identifying open file::Used by OS for file operations
92::What is access control list?::Defines permissions for users::Enhances file security
93::What is protection domain?::Set of resources accessible to a process::Ensures controlled access
94::Difference between protection and security?::Protection controls access, security prevents threats::Security includes authentication
95::What is authentication?::Verifying user identity::Ensures authorized access
96::What is authorization?::Granting access rights::Determines allowed actions
97::What is system call?::Interface between user and kernel::Allows user programs to request OS services
98::Examples of system calls?::Fork, exec, read, write::Used for process and file management
99::What is fork system call?::Creates new process::Child process is duplicate of parent
100::What is exec system call?::Replaces process image::Used to load new program
101::What is zombie process?::Terminated but not removed from process table::Occurs when parent doesn't collect exit status
102::What is orphan process?::Parent terminated before child::Child is adopted by init process
103::What is daemon process?::Background process::Provides system services
104::What is load average?::Average number of runnable processes::Indicates system load
105::What is interrupt?::Signal to CPU for immediate attention::Used for I/O and exceptions
106::Hardware vs Software interrupt?::External vs internal signal::System calls are software interrupts
107::What is polling?::CPU checks device status repeatedly::Inefficient compared to interrupts
108::What is DMA?::Direct Memory Access::Allows devices to access memory directly
109::Advantage of DMA?::Reduces CPU load::Improves I/O performance
110::What is device driver?::Software controlling hardware device::Acts as interface between OS and hardware
111::Character vs Block device?::Stream vs fixed-size blocks::Examples: keyboard vs disk
112::What is buffering?::Temporary data storage::Smooths data transfer
113::What is spooling?::Simultaneous peripheral operations online::Used in printing
114::What is kernel panic?::Critical system error::Forces system halt
115::What is dual-mode operation?::User and kernel modes::Ensures system protection
116::What is trusted computing base?::Core security components::Must be secure for system safety
117::What is virtualization?::Running multiple OS on same hardware::Improves resource utilization
118::What is hypervisor?::Virtual machine manager::Controls virtual machines
119::Type 1 vs Type 2 hypervisor?::Bare-metal vs hosted::Type 1 is faster and secure
120::What is containerization?::OS-level virtualization::Uses shared kernel
121::Difference between VM and container?::VM has full OS, container shares kernel::Containers are lightweight
122::What is bootloader?::Loads OS kernel::Example: GRUB
123::What is init process?::First user-space process::Has PID 1
124::What is runlevel?::System operational state::Controls services
125::What is journaling file system?::Logs changes before committing::Prevents corruption
126::Example of journaling FS?::EXT4::Used in Linux
127::What is power management in OS?::Controlling power usage::Extends battery life
128::What is ACPI?::Advanced Configuration and Power Interface::Manages power states
129::What is sleep state?::Low power mode::RAM powered
130::What is hibernation?::Save memory to disk::System powers off completely
131::What is POSIX?::Portable Operating System Interface::Standardizes Unix systems
132::What is shell?::Command interpreter::Interface between user and OS
133::What is CLI vs GUI?::Text vs graphical interface::GUI is user-friendly
134::What is command interpreter role?::Executes user commands::Starts programs
135::What is scheduling queue?::List of processes in scheduling state::Includes ready and waiting queues
136::What is long-term scheduler?::Controls admission of jobs::Manages degree of multiprogramming
137::What is short-term scheduler?::Selects next process to run::Invoked frequently
138::What is medium-term scheduler?::Handles swapping::Balances memory load
139::What is throughput?::Completed processes per unit time::Higher is better
140::What is turnaround time?::Completion time minus arrival time::Measures total execution duration
141::What is waiting time?::Time spent in ready queue::Affects responsiveness
142::What is response time?::Time until first response::Important in interactive systems
143::What is fairness in scheduling?::Equal CPU opportunity::Prevents starvation
144::What is scalability?::Ability to handle growth::Important for multiprocessor systems
145::What is fault tolerance?::Ability to continue after failure::Critical for servers
146::What is checkpointing?::Saving process state::Used for recovery
147::What is rollback recovery?::Restoring previous state::Handles failures
148::What is resource allocation graph?::Graph showing resource usage::Used to detect deadlock
149::What is safe state?::System can avoid deadlock::Exists safe sequence
150::Why OS is important?::Manages hardware and software efficiently::Without OS, system is unusable
