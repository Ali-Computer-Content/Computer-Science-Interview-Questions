1::What is Digital Forensics?::The process of identifying, collecting, analyzing, and preserving digital evidence.::Digital forensics ensures evidence is legally admissible while uncovering facts from electronic devices.
2::What is the goal of digital forensics?::To discover and present digital evidence accurately.::The goal is to support investigations without altering original data.
3::What is digital evidence?::Data stored or transmitted digitally.::Includes files, logs, emails, metadata, and network traffic.
4::What are the main phases of digital forensics?::Identification, Collection, Preservation, Analysis, Presentation.::These structured phases ensure legal and technical integrity.
5::What is identification in forensics?::Locating potential evidence.::Investigators determine which devices and data are relevant.
6::What is collection in forensics?::Acquiring digital evidence.::Must be done using forensically sound methods.
7::What is preservation in forensics?::Protecting evidence from alteration.::Maintains integrity and admissibility.
8::What is analysis in forensics?::Examining evidence for insights.::Uses tools to reconstruct events.
9::What is presentation in forensics?::Reporting findings.::Evidence must be clearly explained in court.
10::What is chain of custody?::Documented evidence handling record.::Proves evidence was not tampered with.
11::Why is chain of custody important?::Ensures legal admissibility.::Breaks can invalidate evidence.
12::What is forensic imaging?::Bit-by-bit copy of storage media.::Captures all data including deleted files.
13::What is a forensic image hash?::Cryptographic checksum of image.::Used to verify integrity.
14::Common hash algorithms in forensics?::MD5, SHA-1, SHA-256.::Ensure evidence consistency.
15::What is write blocker?::Prevents data modification.::Used during evidence acquisition.
16::What is live forensics?::Evidence collection from running systems.::Captures volatile data like RAM.
17::What is dead forensics?::Analysis of powered-off systems.::More stable but misses volatile data.
18::What is volatile data?::Data lost after power off.::Includes RAM, processes, network connections.
19::What is non-volatile data?::Persistent stored data.::Includes hard drives and flash memory.
20::What is memory forensics?::Analysis of system RAM.::Reveals malware, keys, and running processes.
21::What is swap file?::Disk-based virtual memory.::May contain sensitive remnants.
22::What is pagefile.sys?::Windows virtual memory file.::Useful in memory analysis.
23::What is slack space?::Unused space within clusters.::May contain remnants of deleted data.
24::What is unallocated space?::Disk space not assigned to files.::Often contains deleted data.
25::What is file carving?::Recovering files without metadata.::Uses file signatures.
26::What is file signature?::Unique file header/footer.::Identifies file type.
27::What is metadata?::Data about data.::Includes timestamps and ownership.
28::What are MAC times?::Modified, Accessed, Created times.::Used for timeline reconstruction.
29::What is timeline analysis?::Chronological event reconstruction.::Helps identify attacker actions.
30::What is log analysis?::Reviewing system and application logs.::Reveals user activity and attacks.

33::What is SAM file?::Security Account Manager.::Stores user account information.
